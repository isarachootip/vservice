# คู่มือระบบ VService (VService System Manual)

คู่มือฉบับนี้จัดทำขึ้นเพื่อแนะนำการใช้งาน โครงสร้างระบบ และคุณสมบัติต่างๆ (Features) ของระบบแจ้งซ่อมสินค้าและประสานงาน **VService (Thai Watsadu Service Center)** สำหรับทั้งพนักงานสาขา (User/CS/GR/DC) และผู้ดูแลระบบ (Admin)

## 1. ระบบงานหลัก (Core Features)

### 1.1 หน้าหลัก (Dashboard)
- **การใช้งาน**: แสดงแดชบอร์ดสรุปสถิติตัวเลขภาพรวมงานซ่อมในสาขาที่ผู้ใช้งานสังกัดอยู่ แบ่งตามกลุ่มสถานะต่างๆ เช่น รอดำเนินการ, รอเสนอราคา, เสร็จสิ้น
- **การแยกสิทธิ์**: พนักงานสาขาทั่วไปจะเห็นสถิติเฉพาะสาขาตนเอง ส่วน Admin จะสามารถดูสถิติของทุกสาขาได้

### 1.2 รับเครื่อง / สร้างงาน (Create Repair)
- **การใช้งาน**: หน้าจอลงทะเบียนเปิดใบแจ้งซ่อมใหม่ (ลงทะเบียนอาการเสีย) 
- **จุดเด่น**:
  - ดึงข้อมูลลูกค้าอัตโนมัติเมื่อระบุเบอร์โทรศัพท์
  - ดึงข้อมูลรายละเอียดสินค้า ราคา และทุนจากรหัส SKU หลัก
  - กรณีสินค้าไม่มี SKU ในระบบหลัก สามารถปลดเครื่องหมายถูกที่ช่อง "สินค้าในระบบ" เพื่อกรอก ยี่ห้อ/รุ่น/ประเภทสินค้า ได้อย่างอิสระ
  - แผงอัปโหลดภาพถ่ายประกอบ 4 ช่อง (ภาพด้านบนตัวเครื่อง, ภาพด้านข้าง, ภาพด้านบน/อื่นๆ, และภาพป้าย Serial)

### 1.3 ติดตามงานซ่อมทั้งหมด (Track Status)
- **การใช้งาน**: ตารางแสดงข้อมูลใบงานซ่อมทั้งหมดในระบบเพื่อตรวจเช็คประวัติ ค้นหาด้วยคำค้น (Search) หรือกรองด้วยสถานะ (Status Filters)
- **สิทธิ์การเข้าถึง**: ระบบจัดแบ่งข้อมูลตามสาขา (Location Isolation) พนักงานทั่วไปจะเข้าถึงใบซ่อมที่ไม่ได้จัดทำในสาขาตนเองไม่ได้

### 1.4 ระบบเสนอราคาและการอนุมัติ (Quotations)
- **การใช้งาน**: การรับใบเสนอราคาซ่อมจากทางผู้รับเหมา (Vendor) 
- **ขั้นตอน**: เมื่อ Vendor ประเมินและกรอกค่าบริการรวมถึงราคาอะไหล่เข้ามาในระบบ ใบงานจะปรับเป็นสถานะ "รออนุมัติราคา" พนักงาน CS จะทำการติดต่อลูกค้าเพื่อเสนอราคา และทำการกด **อนุมัติ (Approve)** หรือ **ปฏิเสธ (Reject)** บนหน้าเว็บบนเบสประวัติ

### 1.5 แชตประสานงานระบบ (Internal & Customer LINE Chat)
- **ห้องสนทนาทีมงาน (Staff Chat)**: ห้องแชตภายในสาขาหรือประสานงานระหว่างสาขากับช่าง/Vendor
- **แชตกับลูกค้า (LINE Chat)**: พนักงาน CS สามารถแชตสนทนา คุยตอบโต้กับลูกค้าได้โดยตรงจากระบบ ผ่าน LINE Official Account ของแอปพลิเคชันโดยไม่ต้องใช้โทรศัพท์มือถือส่วนตัว

## 2. ระบบตั้งค่าและหลังบ้าน (Admin Settings & Config)

หน้าจอการจัดการระบบหลักสำหรับผู้ดูแลระบบ (ADMIN) เข้าถึงได้จากเมนูย่อยของแถบ "ตั้งค่า" (Settings) มีองค์ประกอบหลักดังนี้:

| คุณสมบัติ (Feature) | คำอธิบายการตั้งค่า |
|---|---|
| **Status Info** | กำหนดสถานะงานซ่อม และการนับชั่วโมงการทำงาน (SLA Hours) |
| **Vendor Info** | จัดการรายชื่อ ที่อยู่อีเมล และรหัสของผู้รับเหมา (Vendor) |
| **User & Access Info** | จัดการบัญชีผู้ใช้งานระบบ รหัสผ่าน และระดับสิทธิ์ (Roles/Permissions) |
| **Location Info** | จัดการข้อมูลสาขา/คลังสินค้า (Store/DC) รวมถึงรหัสย่อ BU Prefix ของแต่ละแห่ง |
| **Product Info** | คลังข้อมูลสินค้า, ยี่ห้อ, ราคาขาย, และต้นทุนที่เกี่ยวข้อง |
| **Diagnostic Fee Config** | กำหนดค่าตรวจเช็คเบื้องต้น (ค่าเปิดเครื่อง) แยกตามประเภทสินค้า |
| **Margin Config** | กำหนดสัดส่วนกำไรขั้นต่ำ (Margin Floor) และเปอร์เซ็นต์มาร์จินของเครื่องมือ |
| **Service Tier Config** | ตั้งค่าประเภทบริการเร่งด่วน (Normal, Express, VIP) และตัวคูณ SLA |
| **ตั้งค่าประกาศ (Announcements)** | สร้างประกาศแจ้งเตือนแบบวิ่ง Banner ด้านบนของเว็บไซต์ระบบ |
| **ตั้งค่ารูปภาพตัวอย่าง** | กำหนดรูปภาพตัวอย่างประกอบการอัปโหลดไฟล์ในใบแจ้งซ่อม (ดูหัวข้อถัดไป) |

## 3. ระบบจัดการรูปภาพตัวอย่าง (Example Photos Feature)

คุณสมบัติการกำหนด **รูปภาพตัวอย่างและคำแนะนำการถ่ายภาพ (Example Photos)** ถูกพัฒนาขึ้นเพื่อแก้ปัญหาพนักงานอัปโหลดภาพที่ไม่ได้มาตรฐาน (ภาพเบลอ, มุมถ่ายไม่ชัดเจน, ไม่เห็น Serial Number)

### 3.1 สถาปัตยกรรมและที่เก็บข้อมูล (Architecture)
- **ฐานข้อมูล**: จัดเก็บลิงก์รูปภาพและคำอธิบายบนตาราง `system_config` ด้วยคีย์ดังนี้:
  - `example_image_slot1`, `example_image_slot2`, `example_image_slot3`, `example_image_slot4` (เก็บ URL รูปภาพ)
  - `example_desc_slot1`, `example_desc_slot2`, `example_desc_slot3`, `example_desc_slot4` (เก็บข้อความคำแนะนำเพิ่มเติม)
- **การเก็บไฟล์รูป**: เมื่อแอดมินอัปโหลดไฟล์ผ่านหน้าตั้งค่า ไฟล์จะถูกบันทึกลงในเซิร์ฟเวอร์ที่ไดเรกทอรี `/public/uploads/config/`
- **ระบบทำลายขยะ**: หากแอดมินอัปโหลดรูปภาพใหม่แทนที่รูปเดิม หรือกดปุ่ม "ลบรูปตัวอย่าง" ระบบ API หลังบ้านจะทำการสแกนหาไฟล์รูปเดิมในเครื่องและสั่งลบไฟล์ทิ้งทางกายภาพ (unlink) โดยอัตโนมัติ เพื่อป้องกันการสะสมไฟล์ขยะบนเซิร์ฟเวอร์

### 3.2 ขั้นตอนการทำงานสำหรับ Admin (Configuration)
1. ไปที่แถบเมนูด้านซ้าย **ตั้งค่า** -> เลือก **ตั้งค่ารูปภาพตัวอย่าง**
2. ระบบจะดึงข้อมูลรูปภาพและคำอธิบายปัจจุบันมาแสดงในรูปแบบ Grid Card ทั้ง 4 ช่อง
3. แอดมินสามารถคลิกเลือกไฟล์รูปภาพตัวอย่างใหม่ กรอกหรือแก้ไขข้อความคำแนะนำ และกดปุ่ม **บันทึกช่องนี้** เพื่ออัปเดตระบบ
4. หากต้องการยกเลิกการใช้รูปตัวอย่างสำหรับช่องนั้นๆ สามารถกดปุ่ม **ลบรูปตัวอย่าง** ด้านบนของกรอบพรีวิวได้

### 3.3 ขั้นตอนการแสดงผลสำหรับพนักงานสาขา (User Guide)
1. เมื่อพนักงานเปิดหน้าจอ **รับเครื่อง / สร้างงาน** (`/request/add`)
2. ในส่วนการแนบภาพถ่าย ทั้ง 4 ช่องอัปโหลดจะมีปุ่ม **`💡 ดูตัวอย่าง`** ปรากฏขึ้น (เฉพาะช่องที่แอดมินทำการตั้งค่ารูปตัวอย่างไว้)
3. เมื่อคลิกปุ่ม ระบบจะแสดงหน้าต่างป๊อปอัป (Modal Window) โชว์รูปถ่ายที่ถูกต้องและข้อความคำอธิบายประกอบแบบขยายชัดเจน
4. พนักงานปิดหน้าต่างป๊อปอัปแล้วทำการอัปโหลดไฟล์จริงเพื่อส่งใบแจ้งซ่อมตามปกติ

## 4. คำถามที่พบบ่อย (FAQs)

#### Q: จำเป็นต้องเพิ่มช่องเก็บข้อมูลรูปภาพตัวอย่างในตารางของฐานข้อมูลเพิ่มไหม?
**A**: ไม่จำเป็นครับ ระบบใช้ตาราง `system_config` ที่มีอยู่แล้วในการจัดเก็บ Key-Value เป็น string ทำให้ไม่ต้องปรับเปลี่ยน Schema ของ Prisma หรือทำ Database Migration

#### Q: การอัปโหลดรูปตัวอย่างรองรับไฟล์ประเภทใดบ้าง?
**A**: แนะนำให้อัปโหลดไฟล์รูปภาพประเภท `.png`, `.jpg` หรือ `.jpeg` เพื่อการแสดงผลที่รวดเร็วและรองรับการเปิดพรีวิวผ่านเบราว์เซอร์ได้ทันที

#### Q: หากพนักงานแจ้งว่าไม่เห็นปุ่ม "💡 ดูตัวอย่าง" เกิดจากอะไร?
**A**: เกิดจากแอดมินยังไม่ได้ตั้งค่าหรืออัปโหลดรูปภาพตัวอย่างของช่องนั้นๆ ในระบบตั้งค่าหลังบ้าน ให้แอดมินตรวจสอบการตั้งค่าอีกครั้ง
